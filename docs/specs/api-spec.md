## 1. API Design Principles

| Principle | Description |
|-----------|-------------|
| **Type Safety** | Every command has a typed Rust signature, auto-generated to TypeScript via tauri-specta. No `any` types on either side. |
| **Consistent Naming** | Commands use `module_action` format: `container_list`, `llm_proxy_stream`, `mcp_server_start`. |
| **Result-Based Errors** | All commands return `Result<T, AppError>`. Errors are serialized as structured objects. |
| **Minimal Parameters** | Required parameters are explicit. Optional parameters use `Option<T>`. |
| **Streaming via Events** | Long-running operations use Tauri Events with channel-based naming, not invoke responses. |
| **Audit by Default** | Mutating operations are logged to the audit table. |
| **Validation First** | All inputs are validated before execution. |

## 2. tauri-specta Integration

### 2.1 How It Works

`tauri-specta` reads Rust command signatures and generates corresponding TypeScript types and function bindings:

```rust
// Rust command definition
#[tauri::command]
#[specta::specta]
pub async fn container_list(
    state: State<'_, AppState>,
    filters: Option<ContainerListFilters>,
) -> Result<Vec<ContainerInfo>, AppError> { /* ... */ }
```

Generates:

```typescript
// Auto-generated: src/bindings.ts
import { invoke } from '@tauri-apps/api/core';

export async function containerList(
    filters?: ContainerListFilters | null
): Promise<ContainerInfo[]> {
    return invoke('container_list', { filters });
}
```

### 2.2 Type Export Configuration

```rust
// src-tauri/src/main.rs
#[cfg(debug_assertions)]
builder
    .export(
        specta_typescript::Typescript::default()
            .bigint(specta_typescript::BigIntExportBehavior::Number)
            .header("// Auto-generated by tauri-specta. DO NOT EDIT.\n"),
        "../src/bindings.ts",
    )
    .expect("Failed to export TypeScript bindings");
```

### 2.3 Shared Types

All types used in command signatures are defined in `cratebay-core/src/models.rs` with both `serde` and `specta` derives:

```rust
use serde::{Serialize, Deserialize};
use specta::Type;

#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct ContainerInfo {
    pub id: String,
    pub name: String,
    pub image: String,
    pub status: ContainerStatus,
    pub created_at: String,
    pub ports: Vec<PortMapping>,
    pub labels: HashMap<String, String>,
}
```

#### `container_create`

Create a new container.

```rust

    request: ContainerCreateRequest,
```

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `request` | `ContainerCreateRequest` | Yes | Container configuration |

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct ContainerCreateRequest {
    pub name: String,                 // 1-64 chars, alphanumeric + hyphens
    pub image: String,                // e.g., "node:20-alpine"
    pub command: Option<String>,      // override CMD
    pub env: Option<Vec<String>>,     // ["KEY=VALUE", ...]
    pub ports: Option<Vec<PortMapping>>,
    pub volumes: Option<Vec<VolumeMount>>,
    pub cpu_cores: Option<u32>,       // 1-16
    pub memory_mb: Option<u64>,       // 256-65536
    pub working_dir: Option<String>,
    pub auto_start: Option<bool>,     // default: true
    pub labels: Option<HashMap<String, String>>,
    pub template_id: Option<String>,  // use a predefined template
}

#[derive(Debug, Clone, Serialize, Deserialize, Type)]

    pub host_port: u16,
    pub container_port: u16,
    pub protocol: String,   // "tcp" or "udp"
}

#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct VolumeMount {
    pub host_path: String,
    pub container_path: String,
    pub read_only: Option<bool>,
}
```

**Errors:** `AppError::Validation`, `AppError::Docker`

**Example:**


#### `llm_proxy_stream`

Stream a chat completion request through the Rust backend. API keys never leave the backend. Supports three API formats: Anthropic Messages API, OpenAI Responses API, and OpenAI Chat Completions.

The Rust backend automatically selects the correct request format based on the provider's `api_format` field. All requests include dual authentication headers:
- `Authorization: Bearer <apikey>`
- `x-api-key: <apikey>`

```rust
#[tauri::command]
#[specta::specta]
pub async fn llm_proxy_stream(
    app: AppHandle,
    state: State<'_, AppState>,
    channel_id: String,
    provider_id: String,
    model_id: String,
    messages: Vec<ChatMessage>,
    options: Option<LlmOptions>,
) -> Result<(), AppError>
```

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `channel_id` | `String` | Yes | Unique channel for streaming events |
| `provider_id` | `String` | Yes | ID of the configured LLM provider |
| `model_id` | `String` | Yes | ID of the model to use |
| `messages` | `Vec<ChatMessage>` | Yes | Conversation messages |
| `options` | `LlmOptions` | No | Model options |

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct ChatMessage {
    pub role: String,         // "system", "user", "assistant", "tool"
    pub content: String,
    pub tool_calls: Option<Vec<ToolCall>>,
    pub tool_call_id: Option<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct LlmOptions {
    pub model: Option<String>,            // override provider default model
    pub temperature: Option<f32>,         // 0.0 - 2.0
    pub max_tokens: Option<u32>,
    pub top_p: Option<f32>,
    pub tools: Option<Vec<ToolDefinition>>,
    pub reasoning_effort: Option<String>, // "low", "medium", "high" — only used for OpenAI Responses API format
}
```

**Reasoning Effort:** The `reasoning_effort` parameter is only applicable when the provider's `api_format` is `openai_responses`. For other formats (`anthropic`, `openai_completions`), this parameter is ignored by the backend even if provided.

**Returns:** `()` (tokens streamed via events)

**Events emitted:** `llm:stream:{channel_id}` — See [Streaming Events](#4-streaming-events).

**Errors:** `AppError::LlmProxy`, `AppError::NotFound`

#### `runtime_start`

Manually start the built-in container runtime. Triggers detect → provision (if needed) → start → wait for Docker. Called from the Settings page Runtime tab or retry buttons.

**Returns:** `String` — Human-readable status message describing the outcome:
- `"Runtime started and Docker connected"` — Success, Docker is available.

**Behavior:**
1. Detects the current runtime state via `runtime.detect()`.

## 4. Streaming Events

### 4.1 Event Naming Convention

```
{module}:{action}:{channel_id}
```

| Event Pattern | Description |
|---------------|-------------|
| `llm:stream:{channel_id}` | LLM token streaming |
| `exec:stream:{channel_id}` | Container exec output streaming |
| `runtime:health` | Runtime health check updates (no channel, global) |
| `runtime:provision` | Runtime provisioning progress (no channel, global) |

### 4.2 LLM Stream Events

**Event name:** `llm:stream:{channel_id}`

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Type)]
#[serde(tag = "type")]
pub enum LlmStreamEvent {
    /// A token chunk
    Token {
        content: String,
    },

    /// A tool call request from the model
    ToolCall {
        id: String,
        name: String,
        arguments: String,  // JSON string
    },

    /// Stream completed successfully
    Done {
        usage: UsageStats,
    },

    /// Stream error
    Error {
        message: String,
    },
}

#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct UsageStats {
    pub prompt_tokens: u32,
    pub completion_tokens: u32,
    pub total_tokens: u32,
}
```

**Frontend listening:**

```typescript
import { listen } from '@tauri-apps/api/event';

const channelId = crypto.randomUUID();
const unlisten = await listen<LlmStreamEvent>(`llm:stream:${channelId}`, (event) => {
    switch (event.payload.type) {
        case 'Token':
            appendToken(event.payload.content);
            break;
        case 'ToolCall':
            handleToolCall(event.payload);
            break;
        case 'Done':
            onComplete(event.payload.usage);
            break;
        case 'Error':
            onError(event.payload.message);
            break;
    }
});

// Start streaming
await llmProxyStream({ channelId, providerId: 'openai', messages });

// Cleanup when done
unlisten();
```

## 5. Error Response Format

### 5.1 Error Structure

All commands return `Result<T, AppError>`. On the TypeScript side, errors are caught as exceptions:

```typescript
try {
    const result = await containerCreate(request);
} catch (error: unknown) {
    // error is the serialized AppError string
    const message = error as string;
    // e.g., "Validation error: Container name must be 1-64 characters"
    // e.g., "Docker error: connection refused"
    // e.g., "Not found: container with id abc123"
}
```

### 5.2 Error Categories

| Error Type | Prefix | Typical Cause |
|------------|--------|---------------|
| `AppError::Docker` | `"Docker error: ..."` | Docker daemon unavailable, container not found |
| `AppError::Database` | `"Database error: ..."` | SQLite query failure |
| `AppError::LlmProxy` | `"LLM proxy error: ..."` | Provider API error, rate limit |
| `AppError::Validation` | `"Validation error: ..."` | Invalid input parameters |
| `AppError::NotFound` | `"Not found: ..."` | Entity does not exist |
| `AppError::Mcp` | `"MCP error: ..."` | MCP server communication failure |
| `AppError::Runtime` | `"Runtime error: ..."` | VM/runtime issue |
| `AppError::Io` | `"IO error: ..."` | File system error |
| `AppError::PermissionDenied` | `"Permission denied: ..."` | Path traversal, unauthorized action |

### 5.3 Frontend Error Handling Pattern

```typescript
import { toast } from 'sonner';

async function safeInvoke<T>(fn: () => Promise<T>): Promise<T | null> {
    try {
        return await fn();
    } catch (error) {
        const message = error as string;
        if (message.startsWith('Validation error:')) {
            toast.error(message.replace('Validation error: ', ''));
        } else if (message.startsWith('Not found:')) {
            toast.error('Resource not found');
        } else {
            toast.error('An error occurred', { description: message });
        }
        return null;
    }
}

// Usage
const containers = await safeInvoke(() => containerList());
```
